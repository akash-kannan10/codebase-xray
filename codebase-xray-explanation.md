# Codebase X-Ray — Complete Technical Breakdown
Codebase X-Ray is an agentic, browser-based **Developer Intelligence Tool**. Its primary goal is to help developers instantly understand unfamiliar GitHub repositories without cloning them locally. By pasting a public GitHub URL, the app fetches, analyzes, and scores the codebase structure in under 2 minutes.

## Application Architecture Flow
1. **Input Stage (`LandingInput.tsx`)**
   The user provides a GitHub repository URL (e.g., `https://github.com/expressjs/express`).
2. **Analysis Stage (`Index.tsx` & `analyzer.ts`)**
   The main page orchestrates a 6-step analysis process powered by a custom code parser.
3. **Display Stage (`Dashboard.tsx`)**
   The application presents a sophisticated dashboard with two modes (`Beginner` and `Expert`) to visualize the codebase architecture, risk factors, and the optimal onboarding path.

***

## 1. The Core Engine (`src/lib/analyzer.ts` & `src/lib/github.ts`)
The entire application operates entirely in the browser using the public GitHub REST API. It is unauthenticated, relying on intelligent file sampling to stay under rate limits.

### How it fetches data (`github.ts`)
Instead of downloading the source code, the app requests the **Git Tree** object recursively:
- `GET /repos/:owner/:repo/git/trees/:branch?recursive=1`
This returns a flat list of all files and folders in the repository with their paths, sizes, and exact directory structure.

To analyze content without hitting the rate limit, the app strategically selects **at most 30 "important" files** (e.g., `package.json`, `app.ts`, `routes/*`, `controllers/*`) and fetches their specific contents using base64 decoding.

### The 6-Step Analysis Engine (`analyzer.ts`)
The `analyzeRepository` function drives the core logic:

1. **Information Gathering**
   Reads `package.json` to extract the `dependencies` and determine the core framework (Express, Fastify, Next.js, React, Vue, etc.) and programming language (JavaScript vs TypeScript).
2. **Importance Scoring (`scoreFiles`)**
   Ranks the top 30 files in the codebase based on structural importance:
   - **+50 pts**: Is it an Entry Point? (`app.js`, `server.ts`, `main.js`)
   - **+40 pts**: Does it initialize the app? (contains `express()`, `app.listen()`)
   - **+30 pts**: Is it in a core architectural folder? (`controllers/`, `services/`, `models/`)
   - **+20 pts**: Is it heavily imported by other files? (Calculated by matching `import` / `require` statements across the codebase).
3. **Architecture Detection (`detectArchitecture`)**
   Infers the backend/frontend pattern based on folder names:
   - Evaluates combinations of `controllers`, `services`, `models`, `routes`.
   - Returns patterns like **"Layered MVC Style"** (if controllers + services + models exist) or **"Route-Controller Style"** (if routes + controllers exist).
4. **Execution Flow Builder (`buildExecutionFlow`)**
   Generates a dynamic **Mermaid.js** syntax graph mapping the data flow:
   `Client -> App -> Routes -> Middleware -> Controllers -> Services -> Database`.
5. **Risk Analysis (`analyzeRisks`)**
   Scans the repository metadata for common "code smells" and risks:
   - Missing test suites (`tests/`, `spec/`)
   - Missing `README.md`
   - Excessively large files (> 500 estimated vertical lines)
   - Configuration layer absent (`config/`)
   - Dependency bloat (> 30 dependencies).
6. **Onboarding Path (`buildOnboardingPath`)**
   Creates a logical step-by-step reading guide for a new developer:
   Entry Point -> Routes -> Controllers -> Services -> Models -> Settings/Config.

## 2. Frontend User Interface (`src/components/xray/`)
The frontend is built with **React**, **TypeScript**, **Tailwind CSS**, and **shadcn-ui** components.

### Breakdown of the Dashboard
When the analysis finishes, `Index.tsx` swaps the `LoadingScreen` for `Dashboard.tsx`, which lays out the intel across an interactive grid:

- **Top Bar**: Controls the **ModeToggle** (swaps the dashboard between Beginner and Expert verbosity).
- **Sidebar Panels**:
  - `ProjectOverview`: Shows repo stats, detected framework, and language.
  - `RiskPanel`: Highlights the warnings gathered from the Risk Analyzer.
- **Main View**:
  - `ArchitectureSummary`: Displays the inferred architectural pattern (e.g., MVC) and a complexity/coupling rating.
  - `ExecutionFlowDiagram`: Renders the Mermaid.js graph generated by the analyzer.
  - `OnboardingPath`: A vertical timeline component showcasing the best order to read the codebase.
  - `ImportanceTable`: A sortable table listing the most critical files to study, ranked by their importance score.

## Summary
In essence, **Codebase X-Ray** is a static single-page application that uses heuristic pattern matching against the GitHub Tree API to "guess" how a codebase is built. It looks for established naming conventions (like a `controllers` directory or an `index.js` file) to automatically generate comprehensive architecture diagrams and technical documentation on the fly without running a specialized backend.
